# デモ動画シナリオ

このドキュメントは、P TCG Home Gym ManagerのREADME/テックブログ掲載用デモ動画の撮影シナリオです。

## 前提条件と準備

### 推奨事項：事前データ登録

**結論**: デモの見栄えを良くするため、**事前にカードを5～10枚程度登録しておくことを強く推奨**します。

**理由**:
- 進化サーチ機能は、進化系統のカードが複数登録されていないと効果を実感できない
- 検索・フィルタ機能は、データがないと絞り込み結果が空になり訴求力が弱い
- カード一覧画面が空だと「管理アプリ」としての見栄えが悪い

**事前登録推奨カード例** (進化サーチを見せるため):

| カード名   | 進化段階 | タイプ | 進化元     |
| ---------- | -------- | ------ | ---------- |
| ピカチュウ | たね     | 雷     | -          |
| ライチュウ | 1進化    | 雷     | ピカチュウ |
| ヒトカゲ   | たね     | 炎     | -          |
| リザード   | 1進化    | 炎     | ヒトカゲ   |
| リザードン | 2進化    | 炎     | リザード   |
| フシギダネ | たね     | 草     | -          |
| ゼニガメ   | たね     | 水     | -          |

> 💡 この事前登録は「新規登録」のデモで1枚ずつ登録しながら行っても良いですが、
> 動画が長くなりすぎる場合は、事前に登録しておくことを推奨します。

## デモシナリオ

### 動画1: 新規登録 (約30秒)

**目的**: 手動でのカード登録フローを見せる

#### 操作手順

1. **初期画面の表示** (5秒)
   - `http://localhost:8001/` にアクセス
   - ポケモンカードタブが選択されていることを確認
   - 「0件のカードがHITしました」が表示されるはず

2. **新規登録ボタンをクリック** (2秒)
   - ヘッダー右上の「新規カード登録」ボタンをクリック
   - 登録フォームモーダルが開く

3. **フォーム入力** (15秒)
   - 以下のように入力:
     - カード名: `キルリア`
     - 枚数: `3`
     - タイプ: `超` にチェック
     - 進化段階: `1進化` を選択
     - 進化元: `ラルトス`
     - HP: `100`
     - わざのエネルギー: `超` にチェック
     - 弱点: `悪`にチェック
     - 抵抗力: `闘`にチェック
     - にげるコスト: 1
   - 画像は`./demo_manual.JPG`を使用

4. **登録ボタンをクリック** (3秒)
   - 「登録する」ボタンをクリック
   - モーダルが閉じ、一覧にカードが追加される

5. **登録結果の確認** (5秒)
   - 一覧画面にキルリアのカードが追加されていることを確認
   - 枚数「3」と表示されていることを確認

### 動画2: まとめて登録 (約60秒)

**目的**: AI画像解析による一括登録の威力を見せる

#### 事前準備

- 複数のポケモンカードを並べた写真を用意
- 推奨: 3〜5枚のカードがはっきり写った画像

#### 操作手順

1. **まとめて登録ボタンをクリック** (2秒)
   - ヘッダー右上の「まとめて登録」ボタンをクリック
   - アップロードモーダルが開く

2. **画像をアップロード** (10秒)
   - `docs/demo/demo_ai.JPG`を使用
   - ドロップゾーンをクリックして画像を選択
   - または、画像をドラッグ&ドロップ
   - プレビューが表示される

3. **解析を開始** (30秒)
   - 「解析を開始する」ボタンをクリック
   - ローディングインジケーター「AI解析中...」が表示される
   - 解析完了を待つ（約20〜30秒）

4. **プレビュー画面の確認** (10秒)
   - 解析結果がテーブル形式で表示される
   - 各行にカード画像のサムネイル、カード名、属性などが表示される
   - 必要に応じて「編集」ボタンで修正可能
   - `キルリア`の名称をクリックして、既存のカードの枚数を1枚増やし、プレビュー画面で`追加しない`をトグル

5. **一括登録の実行** (8秒)
   - 「すべて登録」ボタンをクリック
   - 登録完了メッセージが表示される
   - モーダルが閉じ、一覧に複数のカードが追加される

---

### 動画3: 検索・フィルタリング (約20秒)

**目的**: 柔軟な検索条件で目的のカードを見つけられることを見せる

#### 操作手順

1. **検索フォームの表示** (3秒)
   - 一覧画面上部の検索フォームを確認

2. **キーワード検索** (7秒)
   - 検索キーワードに `ヤドン` と入力
   - ヤドン関連のカードのみが表示される

3. **フィルタ条件のクリア** (3秒)
   - 「クリア」ボタンをクリック
   - 全カードが再表示される

4. **属性による絞り込み** (7秒)
   - HPのスライドバーで0-100を選択
   - 該当するカードのみが表示される（ヤドン、フラエッテ、など）

---

### 動画4: カード一覧・表示モード切替 (約15秒)

**目的**: 2つの表示モードの違いを見せる

#### 操作手順

1. **カード表示モードの確認** (5秒)
   - デフォルトのカード形式（グリッド表示）を確認
   - 各カードに画像、名前、枚数、タイプバッジが表示されている

2. **テーブル表示への切替** (5秒)
   - 画面上部の表示切替ボタン（リストアイコン）をクリック
   - テーブル形式の一覧表示に切り替わる
   - 一画面でより多くの情報が確認できる

3. **カード表示に戻す** (5秒)
   - 表示切替ボタン（グリッドアイコン）をクリック
   - カード形式に戻る

---

### 動画5: 進化サーチ (約15秒)

**目的**: 進化系統を一覧で確認できる機能を見せる

#### 事前条件

- ヒトカゲ、リザード、リザードンなど進化系統が登録されていること

#### 操作手順

1. **対象カードの特定** (3秒)
   - 一覧からヤドキングを見つける

2. **進化サーチボタンをクリック** (5秒)
   - カード右上の「進化サーチ」ボタンをクリック
   - モーダルが開く

3. **進化系統の確認** (7秒)
   - 「たね」セクションにヤドン
   - 「1進化」セクションにヤドキング
   - 各セクションにカード情報がテーブル表示される

---

### 動画6: 公式サーチ (約10秒)

**目的**: ポケモンカード公式サイトへのリンク機能を見せる

#### 操作手順

1. **対象カードの特定** (2秒)
   - 一覧から任意のカード（例: フラエッテ）を見つける

2. **公式サーチボタンをクリック** (3秒)
   - カード右上の「公式サーチ」ボタンをクリック
   - 新しいタブが開く

3. **公式サイトの確認** (5秒)
   - ポケモンカード公式サイトの検索結果ページが表示される
   - 該当カードの公式情報が確認できる

## 録画時の注意点

### 技術的な注意

- **解像度**: 1920x1080 推奨（READMEでは縮小表示されるため）
- **マウス操作**: ゆっくり、大きく動かす（視認性向上）
- **待機時間**: 各操作後、1〜2秒の余裕を持つ

### コンテンツ的な注意

- **エラー回避**: AI解析は必ず成功する画像を使用
- **データ確認**: 進化サーチ用のデータが事前登録されているか確認
- **表示確認**: 動画開始前に画面を確認し、不要なデータがないことを確認

## 動画作成のワークフロー

1. **事前データ登録** (5分)
   - 推奨カード7枚を手動登録

2. **各動画の録画** (約10分)
   - browser_subagentで各シナリオを実行・録画

3. **出力ファイルの確認**
   - artifacts ディレクトリに WebP 形式で保存される

4. **必要に応じて編集ツールで調整**
   - GIF変換、トリミングなど
