# GitHub機能の使い分けガイド

本プロジェクトにおける、GitHubの各機能の役割と使い分けの指針です。

## 機能別役割一覧

| 機能             | 役割・イメージ   | 具体的な用途                                                                                                                        |
| :--------------- | :--------------- | :---------------------------------------------------------------------------------------------------------------------------------- |
| **README.md**    | **表紙・チラシ** | ・プロジェクトの概要（何ができるアプリか）<br>・最初のセットアップ方法（まずはこれを読む場所）                                      |
| **Issues**       | **タスク管理表** | ・「バグがある」（報告）<br>・「この機能を実装する」（作業宣言）<br>・**やることが明確になったもの**を管理する場所。                |
| **Discussions**  | **掲示板・広場** | ・「これどうやるの？」（Q&A）<br>・「こんな機能あったらいいな」（アイデア出し）<br>・**結論が出ていない、着手前の会話**をする場所。 |
| **GitHub Pages** | **取扱説明書**   | ・詳細なドキュメント（`docs/` 配下の中身）<br>・アーキテクチャ解説、API仕様書など<br>・**読む人:** ユーザーや、深く知りたい開発者   |
| **Wiki**         | **共有ノート**   | 本PJではPagesを主とするため、OFFとします。                                                                                          |


## 運用フロー

1.  **Discussion** で「こんな機能どうかな？」とふわっと話す。
2.  「よし、やろう！」と決まったら **Issue** に切り出す。
3.  開発が終わったらIssueを閉じる。
